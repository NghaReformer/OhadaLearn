# Contexte SYSCOHADA

## La zone OHADA

L'Organisation pour l'Harmonisation en Afrique du Droit des Affaires (OHADA) couvre 17 États d'Afrique francophone. Le **SYSCOHADA** (révisé en 2018, applicable depuis 2019) est le référentiel comptable obligatoire dans toute la zone.

Pour le rapprochement bancaire, les comptes SYSCOHADA pertinents sont :

| Code | Intitulé | Utilisé pour |
|---|---|---|
| 411 | Clients | Chèques clients impayés |
| 472 | Comptes d'attente | Provision en attendant reclassement |
| 521 | Banque | Le compte bancaire à rapprocher |
| 627 | Services bancaires et assimilés | Frais de services bancaires |
| 771 | Intérêts reçus | Produits d'intérêts |

## Les deux systèmes : Normal et Minimal

SYSCOHADA prévoit deux systèmes de présentation :

- **Système Normal** — pour les entités au-dessus des seuils de chiffre d'affaires. Exige un jeu d'états complet incluant le Tableau des Flux de Trésorerie.
- **Système Minimal de Trésorerie (SMT)** — pour les très petites entités. Comptabilité de trésorerie uniquement, avec écritures simplifiées.

Le rapprochement bancaire est exigé dans les deux systèmes. En SMT, c'est *le* contrôle principal sur la trésorerie ; en Système Normal, il alimente le tableau des flux et soutient l'opinion d'audit.

## Attente du contrôle interne

Les guides d'audit OHADA considèrent le rapprochement bancaire mensuel comme un contrôle de base. Les auditeurs vont :

1. Vérifier que les rapprochements sont effectués mensuellement dans des délais acceptables.
2. Examiner les éléments de plus de 60 jours pour des preuves de suivi.
3. Refaire de manière indépendante un échantillon de rapprochements.

Les chèques non encaissés anciens (au-delà de la durée de validité — généralement 1 an) et les éléments non identifiés persistants sont des signaux d'alerte qui aggravent le risque d'audit.
